Keke's Calculator

About this app

  • INSTRUCTIONS

    • This calculator could be manipulated with either a 3-key or 5-key switch control:

    a. A + D + S = selectPrevious + selectNext + clickSelectedItem

    b. Arrowkeys (Left + Right + Up + Down) + Enter = selectPrevious + selectNext + selectUp + selectDown + clickSelectedItem

  • TEST DESCRIPTIONS

    • I tested the above two conditions to compare user performance on equations using a 3-key switch control versus using a 5-key control.

    For each of the conditions, I performed 3 trials to finish the following eqution tasks respectively:

    a. 129 x 30 - 2556

    b. (1456 + 4) / 4

    c. 1233

    Problems:

    a. The 1st condition was not as intuitive as the 2nd one due to the unfamiliarity with this mapping.

    It took me more time to get myself acquainted with the switch control in the 1st condition than the 2nd one even though it had fewer keys.

    b. When the spaceBar was pressed, the page will push down a little bit which made it really hard to finish the task , so I changed the spaceBar control to "s".

  • TEST RESULTS

    • Condition1:

    a. Average Time -- (29 + 19 + 17) / 3 = 21.66s

    b. Average Number of Errors -- 5

    Condition2:

    a. Average Time -- (20 + 20 + 14) / 3 = 18s

    b. Average Number of Errors -- 2

    Conclusions:

    a. Condtion1 generally takes longer time to finish and yields a higher error rate, while condition2 has better performance on both time and accuracy.

    b. The number ofkeys matters in a swicth control setting, where less keys might be more difficult for the user to navigate.

    c. Users' familiarity with the current keyboard mapping could be another cause, users tend to be more familiar with the arrow keys navigation than with the radomly mapped keys.
